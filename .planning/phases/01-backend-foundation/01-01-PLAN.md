  <action>
    **Monorepo setup:**
    - Create `backend/` directory with `pyproject.toml` using `uv init` or manual creation
    - Dependencies: fastapi, uvicorn[standard], fastf1, pandas, numpy, python-dotenv, pydantic-settings
    - Dev dependencies: pytest, httpx, pytest-asyncio
    - Add `backend/cache/` to `backend/.gitignore` and to root `.gitignore`
    - Python 3.12+ target

    **Pydantic models (backend/models/schemas.py):**
    - `EventSummary`: round (int), name (str), country (str), is_cached (bool)
    - `SessionTypeInfo`: key (str — e.g. "Race", "Qualifying"), name (str)
    - `LapData`: LapNumber (int|None), Driver (str), LapTime (float|None — seconds), Time (float|None — session timestamp seconds), PitInTime (float|None), PitOutTime (float|None), Compound (str|None), TyreLife (float|None), Position (int|None), Stint (int|None)
    - `ProgressEvent`: pct (int), stage (str)

    **FastF1 service (backend/services/fastf1_service.py):**
    - `get_completed_events(year: int) -> list[dict]`: Uses `fastf1.get_event_schedule(year, include_testing=False)`, filters by `EventDate < datetime.now(timezone.utc)`, returns list of dicts with round, name, country
    - `get_session_types(year: int, event: str) -> list[dict]`: Gets event, inspects Session1-5 fields to return available session types
    - `serialize_timedelta(td) -> float|None`: Converts pd.Timedelta to total_seconds(), returns None for NaT
    - `serialize_laps(laps: pd.DataFrame) -> list[dict]`: Converts every field to Python primitive per Pattern 3 in RESEARCH.md. Fields: LapNumber, Driver, LapTime, Time, PitInTime, PitOutTime, Compound, TyreLife, Position, Stint
    - `load_session_stream(year, event, session_type, app) -> AsyncIterable[ServerSentEvent]`: Acquires per-session asyncio.Lock from app.state.session_locks, yields SSE progress events at named stages (5% connecting, 20% fetching session, 50% loading laps, 80% processing), runs `session.load(laps=True, telemetry=False, weather=False, messages=False)` via `asyncio.to_thread`, yields complete event with serialized laps data
    - CRITICAL: Always use `asyncio.to_thread` for `fastf1.get_session` and `session.load` — never call blocking FastF1 functions directly in async code
    - CRITICAL: Convert ALL pandas/numpy types before serialization — Timedelta, numpy.float64, NaT will cause silent 500s

    **Cache service (backend/services/cache_service.py):**
    - `CACHE_DIR`: Path to `backend/cache/`
    - `is_session_cached(year, event, session_type) -> bool`: Best-effort filesystem check for existing cache files (see Pattern 5 in RESEARCH.md)

    **Schedule router (backend/routers/schedule.py):**
    - `GET /api/schedule/{year}` — returns list of EventSummary (completed events for given year, with is_cached flag)
    - `GET /api/schedule/{year}/{event_name}/session-types` — returns list of available session types for the event
    - Year validation: 2018 through current year

    **Sessions router (backend/routers/sessions.py):**
    - `GET /api/sessions/load?year=X&event=Y&session_type=Z` — returns EventSourceResponse streaming progress events then complete event with laps data
    - Uses `load_session_stream` from fastf1_service

    **FastAPI app (backend/main.py):**
    - Lifespan: create cache dir, call `fastf1.Cache.enable_cache(str(CACHE_DIR))`, initialize `app.state.session_locks = {}`
    - Mount schedule and sessions routers with `/api` prefix
    - Add CORS middleware (allow localhost origins for dev)

    **Frontend skeleton (done minimally — plan 02 builds UI):**
    - Run `npm create vite@latest frontend -- --template react-ts` (or create manually if in existing dir)
    - Run `npx shadcn@latest init -t vite` in frontend/
    - Run `npx shadcn@latest add select progress button` for needed components
    - Install: `npm install zustand @microsoft/fetch-event-source`
    - Configure vite.config.ts with `/api` proxy to `http://localhost:8000`
    - Verify frontend builds: `cd frontend && npm run build`
