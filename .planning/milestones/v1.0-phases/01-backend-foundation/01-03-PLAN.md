<!-- From plan 01 backend — the API contracts the frontend must consume -->

Backend API endpoints:
- GET /api/schedule/{year} -> EventSummary[] where EventSummary = { round: number, name: string, country: string, is_cached: boolean }
- GET /api/schedule/{year}/{event_name}/session-types -> SessionTypeInfo[] where SessionTypeInfo = { key: string, name: string }
- GET /api/sessions/load?year=X&event=Y&session_type=Z -> SSE stream:
  - event: "progress", data: { pct: number, stage: string }
  - event: "complete", data: { laps: LapData[] }
  - event: "error", data: { message: string }

LapData = {
  LapNumber: number | null,
  Driver: string,
  LapTime: number | null,  // seconds
  Time: number | null,     // session timestamp seconds
  PitInTime: number | null,
  PitOutTime: number | null,
  Compound: string | null,
  TyreLife: number | null,
  Position: number | null,
  Stint: number | null
}
</interfaces>

  <action>
    **TypeScript types (frontend/src/types/session.ts):**
    - `EventSummary`: round, name, country, is_cached (matches backend schema)
    - `SessionTypeInfo`: key, name
    - `LapRow`: LapNumber, Driver, LapTime, Time, PitInTime, PitOutTime, Compound, TyreLife, Position, Stint (all number|null or string|null)
    - `LoadingStage`: 'idle' | 'loading' | 'complete' | 'error'

    **Zustand store (frontend/src/stores/sessionStore.ts):**
    Follow Pattern 6 from RESEARCH.md exactly:
    - State: year (number|null, defaults to current year), event (string|null), sessionType (string, defaults to "Race"), stage (LoadingStage), progress (number), stageLabel (string), laps (LapRow[]), error (string|null), isCompact (boolean, defaults to false)
    - Actions: setYear, setEvent, setSessionType, setProgress, setLaps, setError, reset, toggleCompact
    - setYear resets event and sessionType to defaults
    - setEvent resets sessionType to "Race"
    - setLaps sets stage to 'complete', progress to 100, isCompact to true
    - setError sets stage to 'error'

    **API client (frontend/src/lib/api.ts):**
    - `fetchSchedule(year: number): Promise<EventSummary[]>` — GET /api/schedule/{year}, typed response
    - `fetchSessionTypes(year: number, event: string): Promise<SessionTypeInfo[]>` — GET /api/schedule/{year}/{event}/session-types, typed response
    - Both throw on non-200 responses with meaningful error messages

    **SSE client (frontend/src/lib/sse.ts):**
    - `loadSession(year: number, event: string, sessionType: string, store: SessionStore): void`
    - Uses `@microsoft/fetch-event-source` (NOT native EventSource — see Pitfall 5 in RESEARCH.md)
    - On "progress" event: call store.setProgress(pct, stage)
    - On "complete" event: parse laps from data, call store.setLaps(laps)
    - On "error" event: call store.setError(message)
    - On connection error: call store.setError with network error message
    - Set `openWhenHidden: true` to prevent reconnect when tab is backgrounded during long loads
