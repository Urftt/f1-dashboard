  <files>frontend/src/components/GapChart/GapChart.tsx</files>
  <action>
Create `frontend/src/components/GapChart/GapChart.tsx`:

1. Import `Plot` from `react-plotly.js`, `useGapData` from `./useGapData`, and `useSessionStore` for `currentLap`.

2. The component renders a `<Plot>` with:
   - `data`: The `segments` array from `useGapData()` — each segment is already a valid Plotly trace object with team-colored lines
   - `layout` using Plotly dark template per user decision:
     - `template: 'plotly_dark'`
     - `paper_bgcolor: 'transparent'`, `plot_bgcolor: 'transparent'`
     - `yaxis`: zeroline true, zerolinecolor `'rgba(255,255,255,0.3)'`, zerolinewidth 1.5, gridcolor `'rgba(255,255,255,0.08)'`, title `'Gap (s)'`
     - `xaxis`: gridcolor `'rgba(255,255,255,0.08)'`, title `'Lap'`
     - `margin: { t: 16, r: 8, b: 40, l: 56 }`
     - `showlegend: false`
     - `hovermode: 'x unified'`
     - `shapes`: Vertical dashed replay cursor line at `currentLap`:
       ```
       shapes: currentLap > 0 ? [{
         type: 'line',
         x0: currentLap, x1: currentLap,
         y0: 0, y1: 1,
         yref: 'paper',    // CRITICAL: paper coords to span full chart height
         line: { color: 'rgba(255,255,255,0.6)', width: 1.5, dash: 'dash' }
       }] : []
       ```
   - `useResizeHandler={true}` for responsive sizing
   - `style={{ width: '100%', height: '100%' }}`
   - `config={{ displayModeBar: false }}` — hide Plotly toolbar for clean look

3. Wrap the Plot in a container div with appropriate height (e.g., `h-[400px]` or `min-h-[350px]`).

4. Handle empty state: if `segments.length === 0`, show a subtle message like "Select two drivers to see their gap" (though in practice auto-selection means this is rare).

Key details:
- The `hovertemplate` format `'Lap %{x}: %{y:+.3f}s<extra></extra>'` is already set in each segment from useGapData. The `<extra></extra>` suppresses the trace name box.
- `yref: 'paper'` on the cursor shape is critical — without it the line clips to data range instead of spanning full chart height.
- Do NOT call Plotly imperatively (no `Plotly.react()` or `Plotly.newPlot()`). Use the declarative `<Plot data={} layout={} />` pattern.
