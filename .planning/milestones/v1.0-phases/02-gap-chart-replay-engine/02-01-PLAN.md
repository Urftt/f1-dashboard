<task type="auto" tdd="true">
  <name>Task 2: Create useGapData hook for gap calculation</name>
  <files>frontend/src/components/GapChart/useGapData.ts</files>
  <behavior>
    - Given laps with two drivers sharing laps 1-5, returns lapNumbers [1,2,3,4,5] and gaps as timeB - timeA for each
    - Positive gap means Driver A ahead (lower session time = completed lap sooner)
    - Laps where either driver has null Time or null LapNumber are excluded
    - Returns empty arrays when either driver is null
    - Returns segments array where each segment has x, y, color based on who is leading (sign changes create new segments)
    - Segments overlap by 1 point at crossover to avoid visual gaps in the chart line
  </behavior>
  <action>
Create `frontend/src/components/GapChart/useGapData.ts`:

1. Import `useMemo` from React and `useSessionStore` from the store.

2. Export a `useGapData()` hook that:
   - Reads `laps` and `selectedDrivers` from the store via selectors
   - Uses `useMemo` to compute gap data, only recomputing when laps or selectedDrivers change
   - Returns `{ lapNumbers: number[], gaps: number[], segments: PlotlyTrace[] }` where:
     - `lapNumbers` and `gaps` are the raw data arrays
     - `segments` is an array of Plotly trace objects segmented by who is leading (for dynamic line color)

3. Gap calculation algorithm:
   - Filter laps for driverA and driverB separately, excluding rows with null LapNumber or null Time
   - Build a Map<lapNumber, time> for each driver
   - Find shared lap numbers (laps both drivers completed), sorted ascending
   - For each shared lap: `gap = timeB - timeA` (positive = A ahead because A has lower session elapsed time)

4. Segment algorithm (for dynamic team color line):
   - Walk through the gaps array, detecting sign changes (crossing the zero line)
   - Each contiguous run with the same leader becomes one segment
   - Each segment is a partial Plotly trace: `{ x: number[], y: number[], type: 'scatter', mode: 'lines', line: { color, width: 2 }, hovertemplate, showlegend: false }`
   - Color comes from `getDriverColor(driverA)` when gap >= 0 (A leading), `getDriverColor(driverB)` when gap < 0
   - Segments overlap by 1 point at boundaries to avoid visual gaps at crossover points

5. Also export a `useDriverList()` hook that:
   - Reads `laps` from store
   - Returns drivers from the session, derived from unique `Driver` values in laps
   - Groups by team using `DRIVER_TEAMS` lookup
   - Orders within each team by lap 1 grid position (Position at LapNumber === 1)
   - Returns `{ teams: Array<{ team: string, drivers: string[] }> }` — NEVER sort by final position (spoiler)

Note: Since there is no frontend test runner configured, verification is TypeScript compilation only. The gap calculation logic is straightforward enough (filter, map, subtract) that type safety plus manual smoke testing in Plan 04 is sufficient.
