<task type="auto">
  <name>Task 1: Create useReplayTimer hook</name>
  <files>frontend/src/hooks/useReplayTimer.ts</files>
  <action>
Create `frontend/src/hooks/useReplayTimer.ts`:

1. The hook reads `isPlaying`, `replaySpeed`, `currentLap` from the store and calls `setCurrentLap` and `setIsPlaying`.

2. Compute `maxLap` from the laps in the store: `Math.max(...laps.map(l => l.LapNumber ?? 0))`. Memoize this computation.

3. Use a `useRef` to track `currentLap` inside the interval closure — this is CRITICAL to avoid the stale closure pitfall where setInterval always reads the initial currentLap value. Update `lapRef.current = currentLap` on every render.

4. `useEffect` that runs when `isPlaying` or `replaySpeed` changes:
   - If not playing or currentLap >= maxLap, return (do nothing)
   - Compute `intervalMs = 1000 / replaySpeed` (so 1x = 1s/lap, 4x = 250ms/lap)
   - Start `setInterval` that on each tick:
     - Reads `lapRef.current` (NOT the state variable)
     - If next lap (current + 1) >= maxLap: set currentLap to maxLap, set isPlaying to false, clear interval
     - Else: set currentLap to next lap
   - Return cleanup function that clears the interval

5. Export the `maxLap` value so ReplayControls can display "Lap X/Y" and set the scrubber range.

6. Return `{ maxLap }` from the hook.

Key pitfall prevention:
- Use `useRef` for currentLap inside interval — NOT state variable
- Only include `isPlaying` and `replaySpeed` in the useEffect dependency array (and maxLap for edge case)
- The `setCurrentLap` and `setIsPlaying` store actions are stable (Zustand) so they don't need to be deps
