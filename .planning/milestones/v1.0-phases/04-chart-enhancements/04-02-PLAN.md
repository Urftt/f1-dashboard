    **2. Add pit stop shape builder (for GAP-04):**
    - Read `currentLap` from store (add to hook's store selectors)
    - Read `safetyCarPeriods` from store
    - For each selected driver (driverA, driverB): find laps where `PitInTime !== null` from the `laps` array
    - For each pit lap, build a `Partial<Plotly.Shape>` with:
      - `type: 'line'`, `x0/x1: pitLap`, `y0: 0, y1: 1`, `xref: 'x'`, `yref: 'paper' as const`
      - `line: { color: teamColor, width: 1, dash: 'solid' }` (thinner than cursor's 1.5)
      - `layer: 'above' as any` (on top of SC shading)
      - `label: { text: driverAbbr, textposition: 'top left', font: { color: teamColor, size: 9 } }`
    - **Same-lap double pits:** When both drivers pit on the same lap, offset each line by +/-0.15 laps. Check for collision by building a Set of pit laps per driver, then checking intersection.
    - **Progressive reveal:** Filter to `pitLap <= currentLap` before returning
    - Store each shape's associated lap for filtering (attach `_lap` property or filter during construction)

    **3. Add pit stop hover traces (for "VER pit -- Lap 12" tooltip per user decision):**
    Plotly shapes do NOT support hover tooltips. To implement the locked decision "Hover tooltip shows 'VER pit -- Lap 12' format", create invisible scatter traces that provide hover behavior at each pit stop position.

    For each pit stop (same loop as step 2, filtered by `pitLap <= currentLap`):
    - Create a `Partial<Plotly.PlotData>` scatter trace:
      ```typescript
      {
        x: [pitLap],       // same x as the shape (with same +/-0.15 offset for double pits)
        y: [null],          // null y — Plotly will position the hover at the cursor
        type: 'scatter' as const,
        mode: 'markers' as const,
        marker: { size: 1, opacity: 0 },  // invisible marker
        hovertemplate: `${driverAbbr} pit — Lap ${pitLap}<extra></extra>`,
        showlegend: false,
        // Use yaxis reference to span the full chart height for hover detection:
        yaxis: 'y',
      }
      ```
    - NOTE: Since `y: [null]` may not trigger hover, an alternative approach is to use `y: [0]` with the marker fully transparent (`opacity: 0`). The executor should test both approaches and use whichever produces the hover tooltip. A third fallback: create TWO points per pit `y: [yMin, yMax]` with `mode: 'lines'` and `line: { width: 0, color: 'transparent' }` to create an invisible vertical line trace that captures hover across the full y-range. Use `hoverinfo: 'text'` with `text: ['VER pit — Lap 12', 'VER pit — Lap 12']` instead of hovertemplate in this case.
    - Collect all pit hover traces into a `pitStopHoverTraces` array.

    **4. Add safety car shape builder (for GAP-05):**
    - For each `SafetyCarPeriod` from `safetyCarPeriods` where `period.start_lap <= currentLap`:
      - SC type: `fillcolor: 'rgba(255, 200, 0, 0.18)'` (stronger opacity)
      - VSC type: `fillcolor: 'rgba(255, 200, 0, 0.08)'` (lighter opacity)
      - RED type: `fillcolor: 'rgba(255, 0, 0, 0.25)'` with `line: { color: 'rgba(255,0,0,0.5)', width: 2 }`
      - Build `Partial<Plotly.Shape>` with `type: 'rect'`, `x0: period.start_lap`, `x1: Math.min(period.end_lap, currentLap)` (progressive reveal clamp for active periods), `y0: 0, y1: 1`, `xref: 'x'`, `yref: 'paper' as const`
      - `layer: 'below' as any` (behind gap line traces)
      - `label: { text: period.type, textposition: 'top left' as any, font: { color: 'rgba(255,200,0,0.7)', size: 10 } }`
      - For RED: label font color `'rgba(255,0,0,0.7)'`
      - `line: { width: 0 }` for SC/VSC (no border)

    **5. Wrap both shape computations in `useMemo`:**
    - Dependencies: `[laps, drivers, selectedDrivers, currentLap, safetyCarPeriods]`
    - This can be a separate `useMemo` from the existing gap segments computation, or combined. Keeping separate is cleaner since pit/SC shapes need `currentLap` but gap segments do not.
    - CRITICAL: `currentLap` MUST be in the dependency array. Without it, progressive reveal breaks — backward scrubbing won't hide future events (Pitfall 3 from research).

    **6. Return updated result:**

    The `shapes` array is rendered in order, but z-ordering is controlled by the `layer` property:
    - `layer: 'below'` = behind traces (SC/VSC rects)
    - `layer: 'above'` = in front of traces (pit lines, cursor)

    ```typescript
    shapes: [
      ...scShapes,        // layer: 'below' — SC/VSC shading behind traces
      ...pitStopShapes,   // layer: 'above' — pit lines on top of shading
      ...cursorShape,     // layer: 'above' (implicitly, existing behavior) — replay cursor on top
    ],
    ```

    Per user decision: Z-ordering is SC shading (back) -> pit lines (middle) -> gap line (front) -> cursor (front). Gap lines are Plotly traces (always between 'below' and 'above' layers). Cursor remains the topmost shape.

    **3. Wire pit hover traces into Plot data array:**
    The `pitStopHoverTraces` are invisible scatter traces that provide hover tooltips for pit stop positions. Spread them into the `data` prop of the `<Plot>` component alongside the existing `segments`:

    ```typescript
    />
    ```
    The hover traces are invisible (zero-opacity markers) so they don't affect the visual chart, but they enable Plotly's hover system to show "VER pit — Lap 12" tooltips when the user hovers near a pit stop line.
