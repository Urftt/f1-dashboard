**Then implement (GREEN):**

frontend/src/components/StintTimeline/useStintData.ts:
- Export `Stint` interface: `{ driver: string, stintNumber: number, compound: string | null, startLap: number, endLap: number, tyreLifeAtEnd: number | null }`
- Export `deriveStints(laps: LapRow[]): Stint[]` — groups by `${Driver}::${Stint}`, sorts each group by LapNumber, extracts start/end/compound/tyreLife. Uses last non-null Compound in group.
- Export `computeVisibleStints(stints: Stint[], currentLap: number): Stint[]` — filters to startLap <= currentLap, maps endLap to Math.min(endLap, currentLap)
- Export `computeDriverOrder(laps: LapRow[], drivers: DriverInfo[], currentLap: number): Array<{driver: string, position: number, label: string}>` — finds Position at currentLap for each driver, sorts by position ascending, formats label as "P{pos} {abbr}"
- Export `buildStintTraces(visibleStints: Stint[], driverOrder: Array<{driver: string, label: string}>): Partial<Plotly.PlotData>[]` — returns a single bar trace with orientation 'h', flat arrays for x (stint length = endLap - startLap + 1), y (driver label repeated per stint), base (startLap - 1 for 0-indexed or startLap for 1-indexed x-axis — use startLap since laps are 1-based), marker.color (via getCompoundColor), text (via getCompoundLetter), textposition 'inside', insidetextanchor 'middle', showlegend false. Hover template: "%{text} compound<br>Laps %{base} - %{customdata}<br>extra line break removed"
- Export `useStintData()` hook:
  - Reads laps, drivers, currentLap from useSessionStore (3 separate selectors)
  - Memo 1 `[laps]`: `deriveStints(laps)` — computes ALL stints for entire race
  - Memo 2 `[allStints, drivers, currentLap]`: `computeVisibleStints` + `computeDriverOrder` + `buildStintTraces` — filters and orders for current lap. This is cheap (O(60 stints + 20 drivers)) so currentLap dependency is fine.
  - Memo 3 `[currentLap]`: `makeReplayCursorShape(currentLap)` — cursor shape only
  - Returns `{ traces, cursorShapes, driverOrder, yAxisCategories }` where yAxisCategories is the ordered driver label strings for layout.yaxis.categoryarray
